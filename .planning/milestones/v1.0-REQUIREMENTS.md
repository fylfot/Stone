# Requirements: Stone

**Defined:** 2026-01-26
**Core Value:** Users can accurately track and understand where their time goes across projects with minimal friction.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Core Tracking

- [x] **TRACK-01**: User can start timer from menu bar with one click
- [x] **TRACK-02**: User can stop active timer from menu bar
- [x] **TRACK-03**: User can see elapsed time in menu bar while tracking
- [x] **TRACK-04**: User can switch between projects (auto-stops previous timer)
- [x] **TRACK-05**: User can add time entry manually (retroactive logging)
- [x] **TRACK-06**: User can edit existing time entries (start/end times)
- [x] **TRACK-07**: User is prompted when returning from idle with options to keep/discard idle time
- [x] **TRACK-08**: Timer auto-stops when Mac sleeps or screen locks
- [x] **TRACK-09**: Timer auto-stops when Mac powers off
- [x] **TRACK-10**: Timer data persists across app restarts

### Project Organization

- [x] **PROJ-01**: User can create new project (zone)
- [x] **PROJ-02**: User can edit project name inline
- [x] **PROJ-03**: User can delete project (with confirmation)
- [x] **PROJ-04**: User can assign color to project
- [x] **PROJ-05**: User can organize projects into folders (hierarchy)
- [x] **PROJ-06**: User can create/edit/delete folders
- [x] **PROJ-07**: User can add tags to projects
- [x] **PROJ-08**: User can filter projects by tag
- [x] **PROJ-09**: User can reorder projects within folder

### Reporting

- [x] **REPORT-01**: User can view daily time summary (hours per project)
- [x] **REPORT-02**: User can view weekly time summary
- [x] **REPORT-03**: User can view monthly time summary
- [x] **REPORT-04**: User can see time entry history (searchable list)
- [x] **REPORT-05**: User can view pie chart of time distribution by project
- [x] **REPORT-06**: User can view bar chart of time by day/week
- [x] **REPORT-07**: User can export time data to CSV

### Sync

- [x] **SYNC-01**: User's data syncs automatically via iCloud
- [x] **SYNC-02**: User can see sync status indicator
- [x] **SYNC-03**: Conflicts from concurrent edits are resolved automatically

### UI/UX

- [x] **UI-01**: App runs in menu bar (not Dock)
- [x] **UI-02**: User can open Preferences window
- [x] **UI-03**: User can open Reports window
- [x] **UI-04**: App follows system dark/light mode
- [x] **UI-05**: App launches at login (optional setting)
- [x] **UI-06**: User can configure menu bar display format

## v2 Requirements (Deferred)

### Enhanced Tracking

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Screenshot/activity monitoring | Surveillance anti-pattern, breaks trust |
| Built-in invoicing | Use existing tools, export CSV instead |
| Team/multi-user features | Solo freelancer focus |
| iOS/watchOS apps | macOS only for v1 |
| Automatic time tracking | Manual control preferred, no surveillance |
| Third-party integrations | Native-only approach |
| Pomodoro timer | Different use case, separate app |
| Calendar integration | High complexity, defer to v2+ |
